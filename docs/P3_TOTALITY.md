# P3 Totality Map (Invariant I3) ÔÇö rustc_codegen_clr

> **Invariant I3 ÔÇö "totality with loud failure."** Every reachable MIR construct, intrinsic, `TyKind`, and IR shape is either (a) supported-and-correct, or (b) a documented hard wall that fails the build with a clear message. **Zero reachable silent-wrong paths.** The impossible may refuse to compile; it may never miscompile.
>
> This map is a *reading-level* census (P2 found silent-wrong by differential execution; I3 finds them by reading the arms). A loud `todo!`/`panic!`/`unreachable!` on genuinely unreachable code is **not a bug** ÔÇö it is I3-acceptable. Only paths that compile to wrong behavior without erroring, or loud bails on *reachable* safe-stable code, are findings.

**242 sites scanned** across 8 areas (terminators, intrinsics, rvalue/statement/aggregate, type-lowering, operand/const/statics, place-projections, call/ABI/sig, cilly IR exporters + typechecker).

**The `silent_wrong` enum label massively over-counts the real danger.** Adversarial verification reduced the 24 enum-labeled `silent_wrong` sites to **a small handful of genuine, reachable, defined-divergence miscompiles**. Most "silent_wrong" labels are one of: (a) correct-by-construction with no "correct-silent" enum value available (TypeId arms, the `match (align,len){_=>}` blob shell, conservative `_=>false` defaults); (b) real bugs gated behind **unstable features** (`coerce_unsized`, `repr_simd`); or (c) real bugs requiring a **>4 GiB inline type** that no real program can instantiate.

**P2 overlap:** 18 sites sit inside already-closed P2-S1..S6 / seam-audit surfaces (call terminator, Assert/overflow, SwitchInt, Drop/Terminate, discriminant, PointerExposeProvenance, Indirect-offset statics, vtable, TLS, fn_abi, atomics, float-valuetype, u128 ctlz). Marked `already_covered=p2`, listed for completeness, **not re-worked**.

**Verdict: the codebase is substantially I3-clean.** The faithful-to-MIR translation discipline pays off ÔÇö nearly every catch-all is a loud `todo!`/`panic!` over a structurally-impossible MIR shape, not a benign-looking wrong fallback. The cast machinery (the prime hunting ground for byte-level silent bugs) came up **clean**. The fatal CIL typechecker (`TYPECHECK_CIL=true`, `ALLOW_MISCOMPILATIONS=false` by default) is a real loud backstop that converts several would-be miscompiles into loud build aborts.

### Single highest-priority I3 action

**Ôťů FIXED in P3-S1** (see ┬ž2#1). `static_data.rs:231` ÔÇö `GlobalAlloc::VTable` arm previously registered an uninitialized null `v_{alloc_id:x}` static and returns it as the vtable pointer, with no compile error.** This is the **one confirmed reachable-from-safe-stable silent-wrong** that produces a defined divergence: a **`static OBJ: &dyn T = &S;`** (const/static trait object) routes its vtable relocation through `add_allocation` Ôćĺ this arm Ôćĺ a null vtable, and `OBJ.f()` dispatches through null Ôćĺ NullReferenceException at runtime with **zero diagnostic**.
